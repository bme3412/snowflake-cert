Databases, Tables & Views

How Snowflake organizes data: databases, schemas, table structures, and views. Guides will be added here as you add them from documentation/02-databases-tables-views/.

  • Table structures — Micro-partitions, data clustering, clustering keys, reclustering, and automatic clustering.
  • Temporary & transient tables — Session-based temp tables, transient tables (no Fail-safe), storage, naming, and comparison of types.
  • External tables — Query data in S3, GCS, or Azure without loading; metadata refresh (manual and auto via SQS, Pub/Sub, Event Grid); Hive metastore integration.
  • Hybrid tables — Unistore row store + column store; primary key, indexes, bulk load; AWS and Azure only; cost, cloning, limitations, TSS.
  • Interactive tables — Low-latency interactive queries with interactive warehouses; CLUSTER BY, TARGET_LAG; select AWS regions; limitations, sizing, cost.
  • Tables in Snowsight — Work with tables in the UI: Database Explorer, table details, load data, Copy History, data preview.
  • Search optimization — Search access path for selective lookups; EQUALITY, SUBSTRING, FULL_TEXT, GEO; point lookups, joins, semi-structured, cost.
