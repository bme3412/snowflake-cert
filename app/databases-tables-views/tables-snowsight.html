Working with tables in Snowsight

You can work with tables in SQL or in Snowsight. In Snowsight, use Catalog » Database Explorer to find tables, view details, preview data, load data, and monitor copy history.

Introduction

In Snowsight, open Catalog » Database Explorer, then search for or browse to the table and select it. From there you can explore table and column details, preview data, load data from files, and monitor the table-level Copy History. For SQL commands, see Table, view, & sequence DDL in Snowflake docs.

You must use a role with the relevant table privileges to work with tables in Snowsight.

Explore table details

After opening a table, you can review the table name and its database and schema. You can also see: the type of table, the owner role, when the table was created (hover for exact date/time), row count, and table size (e.g. 2.5 KB).

On the Table Details tab, the Table definition section shows the SQL definition. The Columns tab lists column information; use Search to find columns by name. In the Privileges section you can manage object privileges (see Manage object privileges with Snowsight in Snowflake docs).

Manage a table

Use More options (or equivalent) to perform basic management tasks:

  • Edit — Edit the table name or add a comment.
  • Clone — Clone the table.
  • Drop — Drop the table.
  • Transfer Ownership — Transfer ownership to another role.

To load data into the table from files, select Load Data. You can monitor data loading activity using the table-level Copy History.

Preview data

The Data Preview tab shows up to the first 100 rows of the table. Use More options on the preview to: sort data ascending or descending, change decimal precision, show thousands separators, or display values as percentages. Available options depend on the column data type.

Data preview requires a warehouse. Snowsight uses your default warehouse by default, or you can select a different warehouse.
